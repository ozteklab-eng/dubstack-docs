## 1. Create a site

Start from the dashboard and choose either a connected Git repository or a hosted repository created by DubStack.

## 2. Add content

Write MDX pages and configure navigation in `dubstack.json`.

## 3. Publish

Changes flow through Git, sync to R2, and render on your tenant domain or custom domain.

<Callout type="info">
  The production read path is Git to sync worker to R2 to the docs frontend. Avoid direct content writes outside this flow.
</Callout>

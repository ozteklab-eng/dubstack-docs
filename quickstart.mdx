# Quickstart

Start with a working docs site, make one real customization, and get your first publish out without detouring into setup debt.

## Get started in three steps

Get your documentation running locally and make your first meaningful change.

### Step 1: Set up your local environment

<CardGroup cols={2}>
  <Card title="Clone your docs locally" icon="github">
    Pull your docs repository down so you can edit MDX, config, and assets in one place.
  </Card>
  <Card title="Start the preview server" icon="monitor">
    Run the local preview so you can iterate on content and layout before publishing.
  </Card>
</CardGroup>

### Step 2: Configure your site

<CardGroup cols={2}>
  <Card title="Connect your repository" icon="link">
    Link the repository in the dashboard so saves and publishes flow through the real Git-backed path.
  </Card>
  <Card title="Update site name and colors" icon="palette">
    Adjust your site metadata and theme tokens so the docs reflect your product from the start.
  </Card>
</CardGroup>

### Step 3: Go live

<Card title="Publish your docs" icon="rocket">
  Save your changes, trigger a publish, and confirm the live site reflects your latest content and navigation.
</Card>

DubStack can render API documentation from OpenAPI specs, manually authored MDX endpoint pages, or a mix of both.

Use this section when your docs need endpoint pages, request examples, response schemas, and an interactive playground.

## Supported API documentation paths

| Path | Use it when |
| --- | --- |
| OpenAPI-generated pages | You already have an OpenAPI 3.x JSON spec and want DubStack to generate endpoint pages from it. |
| Manual MDX endpoint pages | You want full control over page copy while still attaching each page to an API operation. |
| Embedded playgrounds | You want to place a runnable endpoint example inside a guide or tutorial. |

## Source of truth

Keep API specs in the same Git repository as the rest of your docs.

The canonical location for the main spec is:

```text
api-reference/openapi.json
```

You can also store named specs under:

```text
api-reference/openapi/payments.json
api-reference/openapi/admin.json
```

After the repo syncs, DubStack stores the spec with the published docs artifacts and uses it to render API reference pages and playgrounds.

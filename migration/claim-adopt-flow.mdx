Claim and adopt promotes a public preview into an owned dubstack site.

This creates a clear boundary between an unclaimed preview and a customer-managed production docs site.

## Lifecycle

```text
Public preview
-> claim request
-> workspace ownership check
-> site creation or adoption
-> source connection
-> production publish
```

## What changes after adoption

| Before claim | After adoption |
| --- | --- |
| Preview lives on `dubstack.wiki`. | Site lives on `*.dubstack.live` or a custom domain. |
| Preview is unclaimed. | Site belongs to a workspace. |
| Source may be generated or imported. | Source is connected to a managed repository or hosted source. |
| Preview can be garbage-collected unless protected. | Customer site follows workspace retention and billing rules. |

## Adoption checklist

Before adopting a preview:

- confirm you control or are authorized to manage the source repository
- review generated or imported content
- decide the production branch
- verify the desired site slug/subdomain
- connect Git or hosted source
- publish and test the live URL

## After adoption

After a preview becomes an owned site:

- configure custom domain if needed
- invite team members
- add redirects from old docs URLs
- review assistant/search behavior
- remove or archive the old preview if it should no longer be discoverable

## Safety boundary

A public preview should never silently become a customer production site. Ownership, source connection, and workspace context must be explicit.

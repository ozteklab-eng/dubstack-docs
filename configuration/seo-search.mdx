Good metadata helps readers, search engines, and AI tools understand your docs.

Start with strong page titles and descriptions, then add redirects and AI-readable surfaces as the docs grow.

## Page metadata

Set `title` and `description` in frontmatter:

```mdx
---
title: Authentication
description: Learn how to authenticate API requests.
---
```

The title should be specific. The description should explain what the reader can do after reading the page.

## Search quality

Improve search by writing pages with:

- descriptive headings
- specific product terms
- short procedural steps
- examples near the explanation they support
- links to related pages

Avoid generic titles such as `Overview` in deep sections unless the navigation context makes the subject clear.

## AI-readable docs

Published docs expose machine-readable surfaces so AI tools can use current docs content:

| Surface | Purpose |
| --- | --- |
| `/llms.txt` | Entry point for AI tools to discover docs content. |
| `/llms-full.txt` | Larger docs index when available. |
| `/mcp` | Public read-only MCP endpoint for search and file-style docs access. |
| `/skill.md` | Skill instructions for supported AI tools. |
| Page Markdown | Markdown representation of docs pages when supported by the renderer. |

## Indexing checklist

- Keep public pages useful without requiring dashboard access.
- Do not publish private credentials or internal-only runbooks.
- Use redirects when changing established URLs.
- Keep OpenAPI summaries and descriptions accurate.
- Republish after large navigation or metadata changes.

## Troubleshooting search

If a page is missing from search or assistant answers:

1. Confirm the page is in navigation or intentionally discoverable.
2. Confirm the page has synced to the live site.
3. Confirm the page is not hidden from public output.
4. Run a manual update.
5. Ask again using terms that appear in the page title or headings.

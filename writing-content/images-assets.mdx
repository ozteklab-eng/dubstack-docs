Images and assets should live with the docs source whenever possible.

This keeps previews, pull requests, and published docs reproducible.

## Recommended structure

```text
images/
  dashboard-preview.png
  editor-navigation.png
  api-playground.png
```

Reference images from MDX:

```mdx
![Dashboard preview](/images/dashboard-preview.png)
```

## Alt text

Write alt text that explains the image content:

```mdx
![The dashboard home page showing the live preview card and recent activity](/images/dashboard-home.png)
```

Avoid alt text like `screenshot` or `image` because it does not help readers or AI tools.

## Frames

Use a frame when the image benefits from a visual boundary:

```mdx
<Frame>
  <img src="/images/editor-navigation.png" alt="Editor navigation pane" />
</Frame>
```

## Asset updates

When replacing an image:

1. Keep the file name stable if the meaning is the same.
2. Use a new file name if the image shows a different product area.
3. Commit the image with the page that references it.
4. Publish and verify the live page.

## Large media

For large videos or files, prefer an external media host and embed the video or link to the file. Keep the docs repository focused on files needed to render pages reliably.

## Checklist

- Use descriptive file names.
- Use compressed assets when possible.
- Add alt text for every image.
- Avoid screenshots that expose secrets, private customer data, internal URLs, or account identifiers.

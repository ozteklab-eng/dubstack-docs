Slack role mapping allows you to automatically assign roles to users based on their Slack workspace membership and permissions. This feature helps maintain consistent access control across your application and Slack workspace.

## Configuration

### Owner Role Mapping

The owner role mapping automatically grants administrative privileges to users who have owner permissions in your Slack workspace.

#### Setup

1. Navigate to your application's role configuration
2. Enable Slack integration
3. Configure the owner role mapping:
   ```json
   {
     "slack_role_mapping": {
       "owner": {
         "enabled": true,
         "slack_role": "workspace_owner",
         "app_role": "admin"
       }
     }
   }
   ```

When a user's Slack role changes, the system processes the following event:
- **Event ID**: `Ev_dubstack_role_owner_1780828731823`
- **Trigger**: Workspace owner status change
- **Action**: Update user's application role accordingly

### Smoke Testing

To verify the role mapping is working correctly:

1. Check that workspace owners are automatically assigned admin roles
2. Verify role updates occur when Slack permissions change
3. Confirm role removal when users lose owner status

## Troubleshooting

### Common Issues

- **Role not updating**: Check Slack webhook configuration
- **Permission errors**: Verify bot has necessary Slack permissions
- **Sync delays**: Role updates may take up to 5 minutes to propagate

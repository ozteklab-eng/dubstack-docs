<parameter name="file_text"># Slack Docs Authoring Smoke Test Setup

This guide explains how to set up a test-only Slack docs authoring smoke test for validating documentation workflows.

## Overview

The Slack docs authoring smoke test validates that documentation changes can be properly authored, reviewed, and published through Slack integration workflows. This test-only setup ensures your documentation pipeline works correctly without affecting production systems.

## Prerequisites

- Slack workspace with admin permissions
- GitHub repository with docs workflow configured
- Node.js 16+ installed locally

## Setup Steps

### 1. Create Test Slack App

1. Go to [Slack API Apps](https://api.slack.com/apps)
2. Click "Create New App" → "From scratch"
3. Name: `docs-authoring-smoke-test`
4. Select your test workspace

### 2. Configure App Permissions

Add the following OAuth scopes under "OAuth & Permissions":

```
- channels:read
- chat:write
- files:read
- users:read
```

### 3. Install App to Workspace

1. Click "Install to Workspace"
2. Copy the Bot User OAuth Token
3. Store token as `SLACK_BOT_TOKEN` environment variable

### 4. Set Up Test Channel

1. Create a dedicated test channel: `#docs-authoring-test`
2. Invite the bot to the channel: `/invite @docs-authoring-smoke-test`

### 5. Configure Environment Variables

Create a `.env.test` file:

```bash
SLACK_BOT_TOKEN=xoxb-your-bot-token
SLACK_CHANNEL_ID=C1234567890  # Your test channel ID
SLACK_EVENT_ID=Ev_dubstack_authoring_pr_success_1780825618877
NODE_ENV=test
```

### 6. Run Smoke Test

Execute the smoke test:

```bash
npm run test:slack-docs-authoring
```

## Expected Behavior

The smoke test should:

1. ✅ Connect to Slack workspace
2. ✅ Post a test message to the designated channel
3. ✅ Trigger docs authoring workflow
4. ✅ Validate PR creation event
5. ✅ Clean up test artifacts

## Troubleshooting

### Bot Not Responding
- Verify bot is invited to test channel
- Check OAuth token permissions
- Ensure `SLACK_CHANNEL_ID` is correct

### Workflow Not Triggering
- Confirm GitHub webhook is configured
- Check event ID format matches expected pattern
- Verify repository permissions

### Test Failures
- Review logs in `tests/logs/slack-smoke-test.log`
- Validate environment variables are set correctly
- Ensure test channel exists and is accessible

## Cleanup

After testing, remove test artifacts:

```bash
npm run cleanup:slack-test
```

This will:
- Delete test messages from Slack channel
- Remove temporary files
- Reset test state

Use this page when a docs site does not look or behave as expected.

## First checks

Start here before changing configuration:

1. Open the dashboard activity feed.
2. Confirm the latest sync completed successfully.
3. Confirm the connected repository and branch are correct.
4. Open the live URL in a fresh tab.
5. Hard refresh the browser.

## Page not found

Check:

- the MDX file exists in the repository
- the page slug is listed in `dubstack.json`
- the file path matches the navigation slug
- the latest commit has synced
- redirects do not point to the missing page

## Navigation is wrong

Check:

- the `navigation` section in `dubstack.json`
- whether the page is hidden
- whether the wrong branch is connected
- whether a manual update is needed

## Theme did not update

Check:

- the theme change was saved
- the configuration change committed to the source
- the latest sync completed
- light and dark logo paths exist
- the browser is not showing stale cached assets

## API reference is missing endpoints

Check:

- the OpenAPI spec path in `dubstack.json`
- whether `pages` lists only selected operations
- whether operations use `x-hidden: true`
- whether methods and paths match the spec exactly
- whether the spec is valid OpenAPI 3.x JSON

## Assistant answer is stale

Check:

- the relevant page is published
- the latest sync completed
- the answer has linked sources
- the page uses clear headings and examples
- a manual update has run after the source change

## Custom domain does not verify

Check:

- DNS points to the target shown in the dashboard
- `/docs` setups proxy both `/docs` and `/docs/*`
- forwarded host/proto headers are preserved for subpath setups
- certificate/DNS propagation has completed

<Callout type="info">
  If the dashboard activity feed shows an infrastructure error, do not keep changing docs content. Wait for the platform path to recover or contact support with the failed activity ID.
</Callout>

Team members let more people work on the same docs workspace.

Invite teammates after the site source and first publish are working.

## Invite a member

1. Open the dashboard.
2. Go to Settings.
3. Open team or members settings.
4. Enter the teammate's email address.
5. Choose the appropriate role.
6. Send the invite.

The invited user should accept with the same email address.

## Role guidance

Assign the least privilege needed:

| Role type | Typical use |
| --- | --- |
| Owner/Admin | Workspace management, billing, integrations, and dangerous settings. |
| Editor | Content and configuration changes. |
| Viewer | Read-only dashboard visibility where available. |

Exact role names can vary by workspace configuration. Use the dashboard role picker as the source of truth.

## Git provider access

Dashboard access and Git provider access are separate.

A teammate may need both:

- dashboard access to open the editor and settings
- repository access to review or merge pull requests
- provider app access if they manage repository installation

## Offboarding

When removing a teammate:

1. Remove dashboard membership.
2. Review Git provider access.
3. Rotate any shared credentials that person could access.
4. Review pending invites and automation ownership.

Billing and plan settings show the workspace's current subscription state and enabled features.

Use this page to understand how plan state affects product behavior.

## What billing controls

Billing can affect:

- custom domains
- team size
- Theme Studio access
- SSO and enterprise controls
- assistant and agent usage
- workflow and integration availability
- support level

The dashboard is the source of truth for your current plan, trial state, and available upgrade paths.

## Trials

Trials can temporarily enable higher-tier features. During a trial, validate the features your team actually needs:

- custom domain setup
- team access
- agent workflows
- assistant behavior
- advanced branding or Theme Studio

Before the trial ends, decide whether the workspace should upgrade or reduce usage to the current plan.

## Feature unavailable

If a feature is locked:

1. Check the plan badge or lock message.
2. Confirm you are in the right workspace.
3. Confirm the feature is not disabled by integration state.
4. Open billing settings to review upgrade options.

## Billing safety

Only trusted workspace admins should manage billing, plan changes, and cancellation actions.

Plan changes can affect public docs behavior when a feature depends on an entitlement.

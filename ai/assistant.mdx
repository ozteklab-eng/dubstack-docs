The assistant helps readers ask questions about your published documentation without leaving the docs site.

It uses the content that has been published for the current site, then returns an answer with sources when relevant.

## What the assistant can do

- Answer setup and usage questions from your docs.
- Link back to source pages so readers can verify details.
- Use the current page as context when opened from a page action.
- Help readers discover related pages when they do not know the exact page name.

## What the assistant cannot do

- Access private dashboard settings from the public docs site.
- Edit files or create pull requests from the public assistant surface.
- Use unpublished local changes that have not synced to the site.
- Bypass visibility, entitlement, or deployment boundaries.

## Improve assistant answers

Write pages so both humans and AI tools can understand them:

- Use descriptive headings.
- Define product-specific terms before using acronyms.
- Keep procedures in ordered steps.
- Add examples for configuration and API usage.
- Cross-link related pages.
- Keep hidden or internal-only material out of public pages.

## Source freshness

The assistant reads from the published docs index. If an answer is missing a recent change:

1. Confirm the change is committed to the connected source.
2. Run a manual update from the dashboard if needed.
3. Wait for the activity item to finish.
4. Refresh the docs page and ask again.

## Configure the assistant

Use [Configure the assistant](/ai/assistant-settings) to manage status, response handling, search domains, bot protection, starter questions, and readiness checks from the dashboard.

## Reader notice

AI responses can contain mistakes. Review linked sources before relying on a generated answer for production changes.

<Callout type="info">
  The dashboard agent is separate from the public assistant. The dashboard agent helps trusted team members draft documentation changes, while the public assistant answers reader questions from published docs.
</Callout>

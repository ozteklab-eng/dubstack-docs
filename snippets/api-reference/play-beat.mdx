Returns playback metadata for a beat pattern including timing data. Audio synthesis occurs client-side in the sequencer visualizer.

> `POST /beats/{id}/play`

### Parameters
- `id` (path, string, required) · Beat pattern ID to play · example: `trap`

### Responses
- `200` Playback metadata with track data and timing.
- `404` Beat pattern not found.

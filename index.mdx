## Product surfaces

| Surface | What it is for |
| --- | --- |
| Dashboard | Manage sites, editor workflows, team access, billing, settings, and publish activity. |
| Editor | Update MDX pages, assets, navigation, and site configuration from the dashboard. |
| Docs shell | Render production documentation sites from Git-backed MDX, OpenAPI specs, and `dubstack.json`. |
| API reference | Publish endpoint docs and browser playgrounds from OpenAPI specs. |
| Assistant | Answer reader questions from published docs with linked sources. |
| Public MCP | Let AI tools search and inspect published docs through a read-only MCP server. |
| Theme Studio | Customize Enterprise docs experiences with visual layout controls. |
| Public wiki previews | Preview public repositories before claim or adoption. |

## Recommended path

1. Create a site from the dashboard.
2. Connect Git or start with a hosted repository.
3. Edit the starter content and navigation.
4. Publish and verify the live docs site.
5. Add API references, assistant settings, and custom domains when the base site is stable.

<Callout type="info">
  The public docs site is powered by the same source-backed publish path your customers use. Changes should be committed, synced, and verified on the live tenant before relying on them.
</Callout>
